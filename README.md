# ECG_Julia

**Atomic and Molecular Calculations with Explicitly Correlated Gaussians: An Efficient Open Implementation in Julia**

📄 **Full paper:** [docs/ECG_Julia_paper.pdf](docs/ECG_Julia_paper.pdf)

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.20751204.svg)](https://doi.org/10.5281/zenodo.20751204)

`ECG_Julia` is an open, modular implementation in the [Julia](https://julialang.org) programming language of the variational method of **explicitly correlated Gaussians (ECGs)** for non-relativistic, finite-nuclear-mass atomic and molecular bound-state calculations — including fully **non-Born–Oppenheimer (non-BO)** calculations in which nuclei and electrons are treated on the same footing.

The code follows the formalism reviewed by Bubin, Pavanello, Tung, Sharkey and Adamowicz [[Bubin2013]](#references) and the matrix-element algebra of Kinghorn [[Kinghorn1996]](#references), Bubin and Adamowicz [[Bubin2008]](#references), and Sharkey and Adamowicz [[Sharkey2014]](#references).

> **The implementation itself is the contribution of this project.** Its three design pillars are a modular matrix-element architecture, a forward-mode automatic-differentiation (AD) energy gradient, and an on-the-fly construction of the permutational-symmetry operator. The benchmark systems below are included to *verify* the implementation, not as new or best-available reference data.

---

## Highlights

- **Four basis-function families**, each a self-contained matrix-element module behind a single uniform interface:
  - `RGL0` — two real correlated Gaussians for $L=0$ states;
  - `CGL0` — two complex correlated Gaussians for $L=0$ states;
  - `N_Pvec` / `N_C_Pvec` — the $L=0$ nitrogen basis with a cubic angular prefactor;
  - `RGL1` — two real correlated Gaussians for $L=1$ states.
- **Exact analytic energy gradient** generated by forward-mode AD ([ForwardDiff.jl](https://github.com/JuliaDiff/ForwardDiff.jl) [[Revels2016]](#references)) — no hand-coded derivatives, no finite-difference cost.
- **On-the-fly permutational symmetry**: the $\hat{Y}^{\dagger}\hat{Y}$ operator is built directly from a human-readable Young-operator string (e.g. `(1+P12)(1-P23)`).
- **Symmetry-term truncation / ramp** that concentrates optimization effort on the dominant permutation terms of large operators (e.g. nitrogen's $7! = 5040$ terms).
- **Reproducible, JSON-driven runs** via a single `ecg_config.json`.
- **Optional text-file bridge** to the reference Fortran program `atom-mol-nonBO` [[ATOMMOLnonBO]](#references) for independent cross-checks.

## Method in brief

For an $n$-pseudoparticle system (after separation of the center-of-mass motion) with internal Cartesian coordinates collected in the $3n$-vector $\mathbf{r}$, the variational wave function is a linear combination of symmetry-projected ECGs,

$$\psi(\mathbf{r})=\sum_{k=1}^{K} c_k\,\hat{Y}\,\phi_k(\mathbf{r}),$$

where $\hat{Y}$ is the permutational-symmetry projector, the $c_k$ are linear coefficients, and the $\phi_k$ are correlated Gaussians whose exponent matrices hold the nonlinear variational parameters. Minimizing the Rayleigh quotient over the $c_k$ gives the generalized symmetric/Hermitian eigenproblem

$$(\mathbf{H}-\varepsilon\mathbf{S})\mathbf{c}=\mathbf{0},\qquad \mathrm{H}_{kl}=\langle \phi_k | \hat{H}\,\hat{Y}^{\dagger}\hat{Y} | \phi_l \rangle,\qquad \mathrm{S}_{kl}=\langle \phi_k | \hat{Y}^{\dagger}\hat{Y} | \phi_l \rangle,$$

solved repeatedly inside an outer optimization of the nonlinear parameters using the analytic energy gradient

$$\frac{\partial\varepsilon}{\partial\alpha}=\mathbf{c}^{\dagger}\left(\frac{\partial\mathbf{H}}{\partial\alpha}-\varepsilon\frac{\partial\mathbf{S}}{\partial\alpha}\right)\mathbf{c}.$$

The four models differ in the form of $\phi_k$ (with $\overline{\mathbf{A}}_k = A_k\otimes I_3$, $A_k=L_kL_k'$):

| Model | Basis function $\phi_k(\mathbf{r})$ |
|---|---|
| `RGL0` | $\exp[-\mathbf{r}'(L_kL_k'\otimes I_3)\mathbf{r}]$ |
| `CGL0` | $\exp[-\mathbf{r}'((A_k+\mathrm{i}B_k)\otimes I_3)\mathbf{r}]$ |
| `N_Pvec` / `N_C_Pvec` | cubic angular prefactor $\times \exp[-\mathbf{r}'\mathbf{A}_k\mathbf{r}]$ (36 elemental integrals) |
| `RGL1` | $z_{m_k}\exp[-\mathbf{r}'(L_kL_k'\otimes I_3)\mathbf{r}]$ |

## Computational cost

For the nitrogen models, each Hamiltonian/overlap matrix element is a sum over the $N_\text{terms}$ permutation terms of the projector ($n! = 5040$ for nitrogen, reduced by the Young operator to a fixed fraction, e.g. 960 or 240 under the truncation ramp). Each term builds the permuted combined width matrix $\tilde A^{(g)} = A_k + P_g' A_l P_g$ and evaluates $|\tilde A^{(g)}|^{-3/2}$, $(\tilde A^{(g)})^{-1}$, and a 36-term angular contraction.

Following the permutational-symmetry scheme of Sharkey & Adamowicz [[Sharkey2014]](#references) (Sec. IV), the `Pvec` variants avoid permutation-matrix products: the forwards $(P_g v = v'P_g')$ and backwards $(P_g'v = vP_g')$ permutations are stored in the two halves of a $2n$-component index vector `pvec`, so a permuted width-matrix element is an **index gather** rather than a matrix multiplication:

$$(P_g A P_g')_{(s,t)} = A_{(\mathrm{pvec}(s),\mathrm{pvec}(t))}.$$

This cuts forming $P_g' A_l P_g$ from $2n^3$ to $n^2$ operations (nitrogen: $686 \to 49$) and permuting a $3n$-vector from $(3n)^2$ to $n$ ($441 \to 7$) — the main per-term saving — while the determinant and inverse stay $O(n^3)$ per term. One matrix element therefore costs $O(N_\text{terms}\, n^3)$, the full $\mathbf{H},\mathbf{S}$ build $O(K^2\, N_\text{terms}\, n^3)$, and the analytic AD gradient adds a constant factor. The symmetry-term truncation/ramp concentrates early optimization effort on the dominant terms.

The helper script [`ECG_Julia/n_pvec_opcount.jl`](ECG_Julia/n_pvec_opcount.jl) prints this breakdown and a scaling table — call `OpCount.report(npart)` (used at the top of the `N_Pvec` / `N_C_Pvec` notebooks).

## Repository structure

| Module | Role |
|---|---|
| `ECG_Param` | Run parameters and defaults (`param0`); the model selector `MatElem_method`; optimization controls (`coeff_threshold`, `coeff_ramp_tol`, `analytic_grad`, trial-generation knobs). |
| `ECG_Config` | Reconstructs an entire run from a single `ecg_config.json` (`setup` / `ECG_Param` / `param0` sections). |
| `ECG_Init` | Data structures, reduced-mass and charge data, transposition matrices, and the $Y$ / $Y^{\dagger}Y$ operators (read from file when present, else built by `SymmetryOperators`). Generates the BFPI table and `set_parvec`. |
| `ECG` | Assembles $\mathbf{H}, \mathbf{S}$ (`compute_H_S`), solves the secular equation (`solve_sec`), generates trials (`stoch_nlp`), runs cyclic optimization (`optim_nlp`, on `Optim.jl`), orchestrates a run (`do_action`). Hosts the truncation/ramp and analytic-gradient accelerators. |
| `ECG_Fortran` | Drives and reads back the reference Fortran program `atom-mol-nonBO` through a plain text-file interface. |
| `ECG_Matelem_*` | Self-contained matrix-element variants, each defining a `module ECG_Matelem` that `ECG.jl` includes according to `MatElem_method`: `ECG_Matelem_RGL0`, `ECG_Matelem_CGL0`, `ECG_Matelem_RGL1`, and the nitrogen variants `ECG_Matelem_N_Pvec`, `ECG_Matelem_N_matr`, `ECG_Matelem_N_C_Pvec`. Each exposes a single `MatrixElements` function returning normalized $\mathrm{H}_{kl}$, $\mathrm{S}_{kl}$ and (optionally) their AD derivatives. |

## Quick start

```julia
using Pkg
Pkg.activate(".")        # use the project environment
Pkg.instantiate()        # install dependencies
```

Each example is driven by a JSON configuration (`ecg_config.json`) and demonstrated in a companion Jupyter notebook (see the table below). Open a notebook with [IJulia](https://github.com/JuliaLang/IJulia):

```julia
using IJulia
notebook(dir = ".")
```

A run is reconstructed from its configuration and executed through `do_action`; the model is selected by `MatElem_method` in `ecg_config.json`.

### Configuration editor

A small [Gradio](https://www.gradio.app) app, [`ecg_config_gradio.py`](ecg_config_gradio.py), lets you load, edit and save an `ecg_config.json` in the browser — editing `setup` as line-based lists, `ECG_Param` / `param0` as key/value tables (so each model's differing fields are preserved), and the optional command script as an editable **actions** table:

```bash
pip install gradio
python ecg_config_gradio.py     # then open the printed local URL
```

### Actions in the configuration (alternate to `inout.txt`)

The command script (`BASIS_ENL`, `OPT_CYCLE`, `FULL_OPT1`, `CHECK`, …) can be carried in the configuration as an optional top-level `"actions"` array instead of the legacy `inout.txt` command script:

```json
"actions": [
  {"Type": "BASIS_ENL", "solver_type": "G", "nfa": 1, "nfo": 5, "ntrials": 500,
   "MaxEnergyEval": 200, "Kstart": 1, "Kstop": 5, "Kstep": 1, "seed": null,
   "nlp0": false, "coeff_nlp": 0.0},
  {"Type": "OPT_CYCLE", "...": "..."},
  {"Type": "CHECK",     "...": "..."}
]
```

Each entry mirrors the `ECG_Init` `action` struct, with `Type` the action **name** and `seed` an integer or `null`. When `"actions"` is present it **replaces** the `inout.txt` action list (the system and basis still come from `inout.txt`); when absent, the legacy `inout.txt` script is used unchanged. Edit it in the Gradio app's **actions** tab, or let `ECG_Config.save_config` write the current `action_list` into the config. After every run, `do_action` also writes the executed actions (with the seeds used) back into the configuration file — just before it writes `inout_1.txt` — leaving the other sections untouched, so the config is ready to drive a reproducible follow-on run.
